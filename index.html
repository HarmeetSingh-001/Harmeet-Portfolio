Pancake Flipper

This project was designed to practice using heuristics to solve searching problems. The pancake problem is, you're given a stack of N pancakes numbered 1-N. The order is random and we want the order to be largest on bottom, smallest on top. You are given a spatula that can be inserted anywhere and flip everything above it. Goal is to use the least number of flips to obtain the desired layout. This solver will take any number of pancakes and return the optimal solution. There is also a Uniform Cost Search (UCS) version implemented solely for the assignment to see how inefficient it is for large problems.

Pancake Zip Instructions for use: Download zip and run the python file.

Knapsack Solver

This project was designed to practice using genetic algorithms to solve search problems. The knapsack problem is, you're given a backpack that can only hold a certain weight, you have 12 items (in our case) and they each have a weight and value. The goal is to get the maximum value while staying in the backpack's weight range. This solver will use a genetic algorithm where each solution will be tested and only the best remain and get offspring to try and make better solutions instead of searching the entire search space slowly.

Knapsack Zip Instructions for use: Download zip and run the python file.

Sudoku Solver

This project was designed to practice using constraints for a search problem. The goal is to be given any sudkou puzzle that has a real solution and find what solution works in an optimal time. This solver will backtrack very early by checking all possible moves after making a change, if at any point a tile loses all possible solutions, we go back and try other values.

Sudkou Zip Instructions for use: Download zip and run the python file.

Papers

BGP Networks

This was a team project and paper. The goal of the project was to utilize network structures to gain new information about some network. My partner and I looked into Border Gateway Protocol networks. That is the main protocol for how the internet works and we wanted to gain 2 bits of insight. 1 was to see if we could predict their tier. A tier is assigned to each autonomous system based on what it is. For example, Tier 3 is given to customers since they pay for services. Tier 1 would be given to the top of the chain to providers who own large networks and systems pay them to get their traffic sent to their destination. The other piece of insight we wanted was to know how much latency we lose out on since a lot of profit is involved with how network traffic is routed. The paper below goes much more in depth into what all the elements of the BGP network are, what we did, and what we found out.
